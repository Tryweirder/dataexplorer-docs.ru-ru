# <a name="azure-data-explorer-data-ingestion-properties"></a>Свойства проглатывания данных Azure Data Explorer 

Прогласивание данных — это процесс, с помощью которого данные добавляются в таблицу и доступны для запроса в Azure Data Explorer. Вы добавляете свойства в команду `with` приема после ключевого слова.

## <a name="ingestion-properties"></a>Свойства приема

В следующей таблице перечислены свойства, поддерживаемые Azure Data Explorer, описаны их и приведены примеры: 

|Свойство              |Описание                                              |Пример                                             |
|----------------------|---------------------------------------------------------|----------------------------------------------------|
|`ingestionMapping`    |Строковое значение, которое описывает сопоставление данных из исходного файла с фактическими столбцами в таблице. Определите `format` значение с соответствующим типом отображения. Дополнительные сведения см. в разделе [о сопоставлении данных](kusto/management/mappings.md).|`with (format="json", ingestionMapping = "[{\"column\":\"rownumber\", \"Properties\":{\"Path\":\"$.RowNumber\"}}, {\"column\":\"rowguid\", \"Properties\":{\"Path\":\"$.RowGuid\"}}]")`<br>(Не рекомендуется: `avroMapping`, `csvMapping`, `jsonMapping`.) |
|`ingestionMappingReference`|Строковое значение, которое описывает сопоставление данных из исходного файла с фактическими столбцами в таблице через именованный объект политики сопоставления. Определите `format` значение с соответствующим типом отображения. Дополнительные сведения см. в разделе [о сопоставлении данных](kusto/management/mappings.md).|`with (format="csv", ingestionMappingReference = "Mapping1")`<br>(Не рекомендуется: `avroMappingReference`, `csvMappingReference`, `jsonMappingReference`.)|
|`creationTime` |Значение времени датирования (отформатированное как строка ISO8601) для использования во время создания просачивания объемов данных. Если этот параметр не указан, используется текущее значение (`now()`). Преодоление значения по умолчанию полезно при глотании старых данных, так что политика удержания будет применяться правильно.|`with (creationTime="2017-02-13T11:09:36.7992775Z")`|
|`extend_schema`|Логическое значение, которое (при наличии) указывает команде расширить схему таблицы (по умолчанию имеет значение `false`). Этот параметр применяется только в командах `.append` и `.set-or-append`. Только допустимые расширения схемы содержат дополнительные столбцы, добавляемые в конец таблицы.|Если исходная схема таблицы, `(a:string, b:int)`допустимое расширение `(a:string, b:int, c:datetime, d:string)`схемы `(a:string, c:datetime)` будет, но не будет действительным|
|`folder` |Для [команд, проглянемых из запроса,](kusto/management/data-ingestion/ingest-from-query.md) папка для присваивания таблице. Если таблица уже существует, это свойство переопределяет папку таблицы.|`with (folder="Tables/Temporary")`|
|`format` |Формат данных [(см. поддерживаемые форматы данных).](ingestion-supported-formats.md)|`with (format="csv")`|
|`ingestIfNotExists`|Строковое значение, которое (при наличии) запрещает считать прием успешным, если в таблице уже есть данные с тегом `ingest-by:` и идентичным значением. Это гарантирует идемпотентность приема данных. Для получения дополнительной информации, [см.](kusto/management/extents-overview.md#ingest-by-extent-tags)|Свойства `with (ingestIfNotExists='["Part0001"]', tags='["ingest-by:Part0001"]')` указывают, что если `ingest-by:Part0001` данные с тегом уже существуют, то не завершаете текущее проглатываемое. Если он еще не существует, это новое проглатывание должно иметь этот набор тегов (в случае, если будущий глотание пытается глотать те же данные снова.)|
|`ignoreFirstRecord` |Логическое значение. Вариант `true` означает, что при приеме следует игнорировать первую запись каждого файла. Это свойство полезно `CSV`для файлов и аналогичных форматов, если первой записью в файле являются имена столбцов. По умолчанию, `false` предполагается.|`with (ignoreFirstRecord=false)`|
|`persistDetails` |Логическое значение, которое (при наличии) указывает, что команда должна сохранять подробные результаты (даже при успешном выполнении), чтобы команда [.show operation details](kusto/management/operations.md#show-operation-details) могла их получить. По умолчанию равен `false`.|`with (persistDetails=true)`|
|`policy_ingestiontime`|Логическое значение, которое (при наличии) указывает, следует ли включить [политику времени приема](kusto/management/ingestiontimepolicy.md) для таблицы, созданной этой командой. Значение по умолчанию — `true`.|`with (policy_ingestiontime=false)`|
|`recreate_schema` |Логическое значение, которое (при наличии) разрешает команде воссоздать схему таблицы. Это свойство относится `.set-or-replace` только к команде. Это свойство имеет приоритет `extend_schema` над свойством, если оба установлены.|`with (recreate_schema=true)`|
|`tags`|Список [тегов,](kusto/management/extents-overview.md#extent-tagging) которые можно связать с просаженными данными, отформатированным как строка JSON |`with (tags="['Tag1', 'Tag2']")`|
|`validationPolicy`|Строка JSON, которая указывает, какие проверки будут выполняться во время приема. [См, где можно проглатчивать данные,](kusto/management/data-ingestion/index.md) чтобы объяснить различные варианты.| `with (validationPolicy='{"ValidationOptions":1, "ValidationImplications":1}')`(это на самом деле политика по умолчанию)|
|`zipPattern`|Используйте это свойство при глотании данных из хранилища, которое имеет архив ЗИП. Это значение строки, указывающее регулярное выражение для использования при выборе файлов в архиве ЗИП для глотания.  Все остальные файлы в архиве будут игнорироваться.|`with (zipPattern="*.csv")`|
