# <a name="visualize-data-with-azure-data-explorer-dashboardspreview"></a>Визуализация данных с помощью панели мониторинга Azure Data Explorer (предварительная версия)

Обозреватель данных Azure — это быстрая и высокомасштабируемая служба для изучения данных журналов и телеметрии. Azure Data Explorer предоставляет веб-приложение для выполнения запросов и создания панелей мониторинга. Панели мониторинга доступны в отдельном веб-приложении, [веб-интерфейсе](web-query-data.md). Кроме того, Azure Data Explorer интегрируется с другими службами панелей мониторинга, такими как [Power BI](power-bi-connector.md) и [Grafana](grafana.md).

Панели мониторинга Azure Data Explorer предоставляют следующие возможности:

* экспорт запросов из веб-интерфейса в панели мониторинга Azure Data Explorer; 
* просмотр данных в пользовательском веб-интерфейсе;
* оптимизированная производительность отображения панели мониторинга.

На приведенном ниже рисунке показана панель мониторинга Azure Data Explorer.

:::image type="content" source="media/adx-dashboards/dash.png" alt-text="Итоговая панель мониторинга":::

> [!IMPORTANT]
> Ваши данные защищены. Панели мониторинга и метаданные пользователей, связанные с панелью мониторинга, шифруются при хранении.

## <a name="create-a-dashboard"></a>Создание панели мониторинга

1. На панели навигации последовательно выберите элементы **Панели мониторинга** и **Новая панель мониторинга**.

    :::image type="content" source="media/adx-dashboards/new-dashboard.png" alt-text="Новая панель мониторинга":::

1. Выберите имя панели мониторинга и щелкните команду **Создать**.

    :::image type="content" source="media/adx-dashboards/new-dashboard-popup.png" alt-text="Создание панели мониторинга":::

## <a name="add-data-source"></a>Добавление источника данных

Добавьте требуемые источники данных для панелей мониторинга.

1. Выберите пункт меню **Источники данных** на панели вверху. Нажмите кнопку **+ Создать источник данных** в области **Источники данных**.

    :::image type="content" source="media/adx-dashboards/data-source.png" alt-text="Источник данных":::

1. В области **Создание нового источника данных**:
    1. Введите **URI кластера** или частичное имя, включая регион, и щелкните команду **Подключить**. 
    1. Выберите из раскрывающегося списка **базу данных** для восстановления.
    1. Используйте значение по умолчанию или при необходимости измените **имя источника данных**. 
    1. Нажмите кнопку **Применить**.

    :::image type="content" source="media/adx-dashboards/data-source-pane.png" alt-text="Панель источника данных":::

## <a name="use-parameters"></a>Использование параметров

Параметры позволяют использовать фильтры панели мониторинга. Параметры значительно улучшают производительность отображения панели мониторинга и позволяют использовать значения фильтров как можно раньше в запросе. Дополнительные сведения о параметрах см. в статье [Использование параметров в панелях мониторинга Azure Data Explorer](dashboard-parameters.md).

1. Щелкните элемент **Параметры** на панели вверху. Щелкните элемент **+ Новый параметр** в области **Параметры**.

    :::image type="content" source="media/adx-dashboards/parameters.png" alt-text="Выбор нового параметра":::

1. Введите значения для всех обязательных полей и нажмите кнопку **Готово**.

    :::image type="content" source="media/adx-dashboards/parameter-pane.png" alt-text="Область параметров":::

|Поле  |Описание |
|---------|---------|
|**Отображаемое имя параметра**    |   Имя параметра, отображаемое на панели мониторинга или на карточке редактирования.      |
|**Тип параметра**    |Это может быть:<ul><li>**Одно выделенное значение.** В качестве входных данных для параметра можно выбрать только одно значение в фильтре.</li><li>**Несколько выделенных значений.** В качестве входных данных для параметра можно выбрать несколько значений в фильтре.</li><li>**Диапазон времени**. Позволяет создавать дополнительные параметры для фильтрации запросов и панелей мониторинга на основе времени. По умолчанию у каждой панели мониторинга есть средство выбора диапазона времени.</li></ul>    |
|**Имя переменной**     |   Имя параметра, используемое в запросе.      |
|**Data type**    |    Тип данных значений параметра.     |
|**Закрепление в качестве фильтра панели мониторинга**   |   Закрепите фильтр на основе параметров на панели мониторинга или открепите его от панели мониторинга.       |
|**Источник**     |    Источник значений параметра: <ul><li>**Фиксированные значения.** Статические значения фильтра предоставляются вручную. </li><li>**Запрос.** Динамические значения предоставляются с помощью запроса KQL.  </li></ul>    |
|**Добавление значения "Выбрать все"**    |   Применимо только к одному и нескольким выделенным значениям. Используется для получения данных для всех значений параметров.      |

## <a name="add-query"></a>Добавление запроса

Функция **Добавить запрос** использует фрагменты языка запросов Kusto для получения данных и отображения визуальных элементов. Каждый запрос поддерживает один визуальный элемент.

1. Щелкните команду **Добавить запрос** на холсте панели мониторинга или в верхней строке меню.

    :::image type="content" source="media/adx-dashboards/empty-dashboard-new-query.png" alt-text="Новый запрос":::

1. В области **Запрос**: 
    1. Выберите источник данных в раскрывающемся списке.
    1. Введите запрос и щелкните элемент **Выполнить**. 
    1. Щелкните команду **+ Добавить визуальный элемент**.

    :::image type="content" source="media/adx-dashboards/initial-query.png" alt-text="Выполнение запроса":::

1. В области **Визуальное форматирование** щелкните элемент **Тип диаграммы**, чтобы выбрать тип визуального элемента. 
1. Присвойте визуальному элементу имя и щелкните элемент **Применить изменения**, чтобы закрепить визуальный элемент на панели мониторинга.

    :::image type="content" source="media/adx-dashboards/add-visual.png" alt-text="Добавление визуального элемента в запрос":::

1. Вы можете изменить размер визуального элемента и щелкнуть элемент **Сохранить изменения**, чтобы сохранить панель мониторинга.

    :::image type="content" source="media/adx-dashboards/save-dashboard.png" alt-text="Сохранение панели мониторинга":::

## <a name="share-dashboards"></a>Совместное использование панелей мониторинга

Используйте меню совместного доступа, чтобы [предоставить разрешения](#grant-permissions) к панели мониторинга, [изменить уровень разрешений пользователя](#change-a-user-permission-level) и [поделиться ссылкой на панель мониторинга](#share-the-dashboard-link).

> [!IMPORTANT]
> Чтобы получить доступ к панели мониторинга требуется следующее:
> * ссылка на панель мониторинга;
> * разрешения для панели мониторинга;
> * доступ к базовой базе данных в кластере Azure Data Explorer.  

1. Выберите пункт меню **Общий доступ** на панели мониторинга вверху.
1. Выберите элемент **Управление разрешениями** в раскрывающемся списке. 

    :::image type="content" source="media/adx-dashboards/share-dashboard.png" alt-text="Раскрывающийся список для предоставления доступа к панели мониторинга":::

### <a name="grant-permissions"></a>Предоставить разрешения

Чтобы предоставить разрешения пользователю, на панели **Разрешения панели мониторинга**:
1. Запишите имя или адрес электронной почты пользователя в поле **Добавить новых участников**.
1. Выберите укажите для параметра **Разрешение** значение **Может просматривать** или **Может изменять**, а затем щелкните элемент **Добавить**.

:::image type="content" source="media/adx-dashboards/dashboard-permissions.png" alt-text="Управление разрешениями панели мониторинга":::

### <a name="change-a-user-permission-level"></a>Изменение уровня разрешений пользователя

Чтобы изменить уровень разрешений пользователя, на панели **Разрешения панели мониторинга**:
1. Используйте поле поиска или прокрутите список пользователей, чтобы найти пользователя.
1. При необходимости измените значение параметра **Разрешение**.

### <a name="share-the-dashboard-link"></a>Предоставление ссылки на панель мониторинга

Чтобы предоставить ссылку на панель мониторинга (одно из двух действий):
* выберите раскрывающийся список **Общий доступ** и щелкните **Скопировать ссылку**;
* в окне **Разрешения панели мониторинга** щелкните элемент **Копировать ссылку**. 

## <a name="enable-auto-refresh"></a>Включение автообновления 

1. Щелкните команду **Изменить** в меню панели мониторинга, чтобы переключиться в режим редактирования.
1. Выберите элемент **Автоматическое обновление**. 
 
    :::image type="content" source="media/adx-dashboards/auto-refresh.png" alt-text="Выбор автоматического обновления":::

1. Установите переключатель в положение **Включено**, чтобы включить автоматическое обновление. 
1. Выберите значения для параметров **Минимальный интервал времени** и **Частота обновления по умолчанию**. 

    :::image type="content" source="media/adx-dashboards/auto-refresh-toggle.png" alt-text="Включение автообновления":::

1. Щелкните команды **Применить** и **Сохранить**, чтобы сохранить панель мониторинга.

> [!NOTE]
> * Выберите минимальный интервал времени, чтобы снизить нагрузку на кластер. 
> * Пользователь, просматривающий панель мониторинга: 
>    * может изменять минимальный интервал времени только для личного использования; 
>    * не может выбирать значение ниже заданного **минимального интервала времени**.
